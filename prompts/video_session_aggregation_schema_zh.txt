你将收到同一原始视频或一次连续视频会话中的多个 2 分钟 window JSON。请聚合成完整 session 级参考证据，输出严格 JSON，不要输出 Markdown，不要输出额外解释。

该输出是隐藏参考标注的一部分，只用于 benchmark 数据构建、标准答案生成、证据追溯和人工审核；被评测 agent 不会看到它。

硬性规则：
1. 所有自然语言字段必须使用中文。
2. 只能根据输入 window JSON 聚合，不要补充外部常识。
3. 单个 session 内仍然不能直接证明“长期习惯”。可以写“本 session 中观察到……”，并把跨 session 验证需求写清楚。
4. 每个结论必须保留 supporting_window_ids。
5. 如果窗口之间存在冲突，不要强行消解，写入 contradictions_or_uncertainties。
6. 输出必须是一个可被 json.loads 解析的 JSON 对象。
7. 上游聚合模型自报的 `confidence` 和 `trackability` 可能已被移除，因为它们未经校准。不得把字段缺失解释为高置信；必须根据 supporting_window_ids、具体证据和不确定性保守判断本层置信度。
