请观看这段第一人称短视频 session，输出严格 JSON，不要输出 Markdown，不要输出额外解释。

你正在为“跨会话长期记忆 benchmark”生成候选证据。此输出不是最终标注，也不是问题生成结果；它会被后续程序聚合、过滤和人工抽查。

核心原则：
1. 只写本 session 中能从画面、可听音频或可读文字直接支持的信息。
2. 单个 session 不能证明“习惯、通常、经常”。只能写“本 session 中观察到……”。
3. 看不清、听不清、无法确认时写入 uncertainties，不要硬猜。
4. 重点提取稳定地点、物体、物体位置、物体状态、状态变化、可复查证据。
5. 不要逐帧罗列。把连续且目标一致的动作合并成事件。
6. 时间使用相对本 session 的 "MM:SS" 或 "MM:SS-MM:SS"。
7. 控制输出长度，优先保证 JSON 完整可解析。

数量上限：
- places 最多 4 个
- objects 最多 10 个
- events 最多 8 个
- state_changes 最多 6 个
- memory_candidates 最多 5 个
- uncertainties 最多 5 个

抽取重点：
1. 优先输出状态变化和最终位置，而不是普通动作摘要。
2. 同类餐具很多时，只保留可追踪或对状态变化重要的物体。
3. 食物种类、包装文字、屏幕内容看不清时不要猜。
4. memory_candidates 只能是候选证据，不要写成用户长期偏好或稳定习惯。
5. 输出必须是一个可被 json.loads 解析的 JSON 对象。
